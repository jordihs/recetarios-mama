# Data Model: Recipe Management

**Feature**: `002-recipe-management` | **Date**: 2026-06-10

Canonical model owned by the Python backend (pydantic schemas + SQLite persistence). The Flutter frontend consumes it as DTOs through the REST contract. All text UTF-8. IDs are UUIDv4 strings generated by the backend.

## Entity overview

```text
Book 1──* Chapter (self-nesting via parent_chapter_id) 1──* Recipe
Book/Chapter ──1 PresentationContent (block list, embedded JSON)
Recipe ──1 introduction blocks, ──1 preparation blocks, ──1 IngredientsList (embedded)
ContentBlock(image) ──ref── ImageFile (content-addressed)
Settings: key/value singleton table
```

## Book

| Field | Type | Constraints |
|-------|------|-------------|
| `id` | string (UUID) | PK |
| `title` | string | required, non-empty, ≤ 200 chars |
| `cover_image` | string (image hash) | optional |
| `presentation` | ContentBlock[] (JSON) | optional, default `[]` |
| `position` | int | order among books, dense from 0 |
| `created_at` / `updated_at` | ISO-8601 UTC | set by backend |

- **Derived**: `description` (first `paragraph` block's plain text; consumers truncate with ellipsis — truncation is a UI concern).
- Deleting a book cascades to its chapters and recipes (after UI confirmation).

## Chapter

| Field | Type | Constraints |
|-------|------|-------------|
| `id` | string (UUID) | PK |
| `book_id` | string | FK → Book, required |
| `parent_chapter_id` | string \| null | FK → Chapter; null = top-level chapter of the book |
| `title` | string | required, non-empty, ≤ 200 chars |
| `cover_image` | string (image hash) | optional |
| `presentation` | ContentBlock[] (JSON) | optional |
| `position` | int | order among siblings (same parent), dense from 0 |
| `created_at` / `updated_at` | ISO-8601 UTC | |

- Nesting: arbitrary depth supported; UI/PDF optimized for ≤ 3 levels (spec assumption). A chapter's parent must belong to the same book; cycles forbidden (enforced in service layer).
- Deleting a chapter cascades to its subchapters and recipes.

## Recipe

| Field | Type | Constraints |
|-------|------|-------------|
| `id` | string (UUID) | PK |
| `chapter_id` | string | FK → Chapter, required |
| `title` | string | required, non-empty, ≤ 200 chars |
| `image` | string (image hash) | optional (recipe's headline photo) |
| `introduction` | ContentBlock[] (JSON) | optional |
| `ingredients` | IngredientsList (JSON) | required (may be empty list for drafts) |
| `preparation` | ContentBlock[] (JSON) | required (may be empty for drafts) |
| `note` | string | optional (legacy NOTA, shown at recipe foot) |
| `position` | int | order within chapter, dense from 0 |
| `created_at` / `updated_at` | ISO-8601 UTC | |

- View order is fixed by FR-014: introduction → ingredients → preparation.
- Edit mode is a UI state; saves are whole-recipe `PUT` (atomic), discard = client drops local draft.

## IngredientsList (embedded in Recipe)

```json
{
  "servings": "6",            // optional string (legacy 'personas' is free text)
  "groups": [
    {
      "title": "Para el aliño",   // optional heading
      "items": ["Aceite de oliva, de 0,4 grados", "Sal y pimienta"]
    }
  ]
}
```

- At least one group; the common ungrouped case is one group with `title: null`.
- Items are free-text strings, order preserved.

## ContentBlock (embedded JSON, discriminated by `type`)

| Type | Payload | Notes |
|------|---------|-------|
| `heading` | `{ "text": string }` | legacy TITULO inside INTRODUCCION |
| `paragraph` | `{ "spans": [{ "text": string, "bold"?: bool, "italic"?: bool }] }` | legacy PARRAFO; spans allow basic inline styling from the rich text editor |
| `image` | `{ "image": hash, "caption"?: string, "placement": "right" \| "block" }` | legacy IMAGEN (placement `right` floats beside following text, matching old XSL) |
| `image_group` | `{ "images": [{ "image": hash, "caption"?: string }], "layout": "grid" \| "row" }` | legacy IMAGENES; the FR-019 grid layout option |
| `table` | `{ "title"?: string, "header": Cell[], "rows": Cell[][] }` with `Cell = { "text"?: string, "image"?: hash }` | legacy TABLA/CABECERA/FILA/CELDA; image cells use text as caption |

- Validation: unknown `type` rejected; `image` hashes must exist in the image store at write time.
- This block list is the single source for screen rendering, editing, and PDF building.

## ImageFile (content-addressed store)

| Field | Type | Constraints |
|-------|------|-------------|
| `hash` | string (sha256 hex) | PK; filename is `<hash>.<ext>` |
| `ext` | string | `jpg` \| `png` \| `gif` \| `webp` |
| `width` / `height` | int | probed at ingest (Pillow) |

- Files live under `<data-dir>/images/`; DB table `images` is the metadata index.
- Ingest paths: recipe/book/chapter image upload, editor insertion, legacy import copy.
- Garbage collection of unreferenced images: explicit maintenance pass at export/import time (not on every delete — YAGNI).

## Settings

| Key | Type | Default |
|-----|------|---------|
| `pdf_output_dir` | string path | OS user folder (resolved by backend at first run) |

Single `settings(key TEXT PRIMARY KEY, value TEXT)` table; extensible for future preferences.

## Search index (FTS5)

```sql
CREATE VIRTUAL TABLE recipe_fts USING fts5(
  title, ingredients, preparation, introduction,
  content='', tokenize = "unicode61 remove_diacritics 2"
);
```

- One row per recipe; columns hold extracted plain text (blocks flattened, spans concatenated).
- Kept in sync by the repository on every recipe insert/update/delete and on import/restore (rebuild).
- Queries: `recipe_fts MATCH ?` with prefix tokens (`term*`), ranked by `bm25`. Results join back to recipes for title + book/chapter breadcrumb (FR-038).

## SQLite schema sketch

```sql
CREATE TABLE books    (id TEXT PRIMARY KEY, title TEXT NOT NULL, cover_image TEXT,
                       presentation TEXT NOT NULL DEFAULT '[]', position INTEGER NOT NULL,
                       created_at TEXT NOT NULL, updated_at TEXT NOT NULL);
CREATE TABLE chapters (id TEXT PRIMARY KEY, book_id TEXT NOT NULL REFERENCES books(id) ON DELETE CASCADE,
                       parent_chapter_id TEXT REFERENCES chapters(id) ON DELETE CASCADE,
                       title TEXT NOT NULL, cover_image TEXT, presentation TEXT NOT NULL DEFAULT '[]',
                       position INTEGER NOT NULL, created_at TEXT NOT NULL, updated_at TEXT NOT NULL);
CREATE TABLE recipes  (id TEXT PRIMARY KEY, chapter_id TEXT NOT NULL REFERENCES chapters(id) ON DELETE CASCADE,
                       title TEXT NOT NULL, image TEXT, introduction TEXT NOT NULL DEFAULT '[]',
                       ingredients TEXT NOT NULL DEFAULT '{"servings":null,"groups":[]}',
                       preparation TEXT NOT NULL DEFAULT '[]', note TEXT,
                       position INTEGER NOT NULL, created_at TEXT NOT NULL, updated_at TEXT NOT NULL);
CREATE TABLE images   (hash TEXT PRIMARY KEY, ext TEXT NOT NULL, width INTEGER, height INTEGER);
CREATE TABLE settings (key TEXT PRIMARY KEY, value TEXT NOT NULL);
-- + recipe_fts virtual table (above) + schema_version pragma/user_version
```

- Foreign keys ON, WAL mode, `user_version` for migrations.
- JSON columns (`presentation`, `introduction`, `ingredients`, `preparation`) validated by pydantic before write; SQLite stores them as TEXT.

## Library archive format (`.recetarios` ZIP)

```text
archive.recetarios
├── library.json     # { "format_version": 1, "exported_at": ISO-8601, "books": [ full nested Book→Chapter→Recipe trees ] }
└── images/          # every image file referenced by library.json (content-addressed names)
```

- Import: validate `format_version` + manifest completeness → build staging DB/images → atomic swap (FR-028 replace semantics).
- Export: serialize live library + copy referenced images; deterministic ordering for diff-ability.

## Legacy → new mapping (import)

| Legacy (JSON per `recetarios-schema.json`) | New model |
|--------------------------------------------|-----------|
| `RECETARIO.TITULO` | `Book.title` |
| `RECETARIO.INTRODUCCION` | `Book.presentation` blocks |
| `CAPITULO` (`@attributes.nombre`) | `Chapter.title`; nested `CAPITULO` → child `Chapter` (hierarchy preserved) |
| `CAPITULO.INTRODUCCION` | `Chapter.presentation` blocks |
| `RECETA.@attributes.imagen` | `Recipe.image` (file copied to store) |
| `RECETA.TITULO` | `Recipe.title` |
| `INGREDIENTES.@attributes.personas` | `ingredients.servings` |
| `GRUPO` (`titulo` attr per XML doc) / `INGREDIENTE[]` | `ingredients.groups[]` / `items[]` |
| `PREPARACION` (string or string[]) | `preparation` paragraph blocks |
| `NOTA` | `Recipe.note` |
| `INTRODUCCION.TITULO / PARRAFO / IMAGEN / IMAGENES / TABLA` | `heading` / `paragraph` / `image` / `image_group` / `table` blocks |
| Image paths `./imgs/...` (relative to document dir) | copied into image store; unresolved → import report entry |
| Single-object-or-array shapes (`oneOf`) | normalized to arrays before mapping |
